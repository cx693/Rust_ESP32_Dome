## 📝 核心代码讲解

### 一、动画原理

动画的本质是 **快速切换静态画面**。人眼在每秒 24 帧以上就会认为是连续运动。

```
帧 1        帧 2        帧 3
  ●           ●
               ●           ●
               ──          ──
球在上方     球下落中     球触地

每帧流程:
  1. 清屏 (fb_clear)
  2. 更新物理状态 (位置、速度)
  3. 碰撞检测与响应
  4. 绘制所有球 (fb_fill_circle)
  5. 刷新到屏幕 (display.flush)
```

### 二、物理模型

#### 重力加速度

```rust
const GRAVITY: i16 = 1;   // 重力加速度 (px/frame²)
const MAX_VY: i16 = 16;   // 最大速度限制（防止穿透）

// 每帧更新速度和位置
ball.vy = (ball.vy + GRAVITY).min(MAX_VY);  // v = v + g·Δt
ball.y += ball.vy;                            // y = y + v·Δt
```

**为什么需要 MAX_VY？**

```
没有速度限制时:
  帧 N:   球在 y=220
  帧 N+1: 球在 y=260  ← 穿过地面 (y=228)！

有了速度限制:
  帧 N:   球在 y=220, vy=16
  帧 N+1: 球在 y=236, vy=16  ← 碰到地面，触发碰撞
```

#### 地面碰撞

```rust
const FLOOR_Y: i16 = HEIGHT as i16 - BALL_RADIUS;  // 地面位置
const GROUND_RESTITUTION: i16 = 75;  // 恢复系数 (75%)
const SETTLE_VY: i16 = 1;            // 静止阈值

if ball.y >= FLOOR_Y {
    ball.y = FLOOR_Y;  // 修正位置

    if ball.vy.abs() > SETTLE_VY {
        // 弹跳: 速度反向并衰减
        ball.vy = -(ball.vy * GROUND_RESTITUTION / 100);
    } else {
        // 速度太小，静止
        ball.vy = 0;
        ball.settled = true;
    }
}
```

**恢复系数的物理含义：**
```
恢复系数 e = 75% 意味着:
  碰撞后速度 = 碰撞前速度 × 0.75
  
  第 1 次弹跳: vy = 10 → 反弹 vy = -7.5
  第 2 次弹跳: vy = 7.5 → 反弹 vy = -5.6
  第 3 次弹跳: vy = 5.6 → 反弹 vy = -4.2
  ...逐渐衰减到静止
```

### 三、球间碰撞（动量守恒）

当两个球的距离小于直径时发生碰撞：

```rust
const BALL_RESTITUTION: i16 = 50;  // 球间恢复系数 (50%)

for i in 0..BALL_COUNT {
    for j in (i + 1)..BALL_COUNT {
        let dx = balls[i].x - balls[j].x;
        let dy = balls[i].y - balls[j].y;
        let dist_sq = dx * dx + dy * dy;  // 距离的平方
        let min_dist = BALL_RADIUS * 2;    // 最小距离 = 两球半径之和

        if dist_sq < min_dist * min_dist && dist_sq > 0 {
            // 简化的动量交换（等质量）
            let vi = balls[i].vy;
            let vj = balls[j].vy;
            balls[i].vy = vj * BALL_RESTITUTION / 100;
            balls[j].vy = vi * BALL_RESTITUTION / 100;

            // 分开两球（沿连线方向推开）
            let dist = isqrt(dist_sq as i32);
            if dist > 0 {
                let overlap = min_dist - dist + 1;
                let nx = dx * overlap / dist / 2;
                let ny = dy * overlap / dist / 2;
                balls[i].x += nx;
                balls[i].y += ny;
                balls[j].x -= nx;
                balls[j].y -= ny;
            }
        }
    }
}
```

**为什么用 dist_sq 而不是 dist？**
```
计算距离需要平方根，而平方根在嵌入式上开销大。
比较距离时，可以比较距离的平方:
  dist_sq < min_dist²  ⟺  dist < min_dist

只在需要实际距离值时才调用 isqrt()。
```

### 四、整数平方根

嵌入式设备没有硬件浮点单元（或很慢），我们用牛顿迭代法求整数平方根：

```rust
fn isqrt(n: i32) -> i16 {
    if n <= 0 { return 0; }
    let mut x = n;
    let mut y = (x + 1) / 2;
    while y < x {
        x = y;
        y = (x + n / x) / 2;  // 牛顿迭代: x' = (x + n/x) / 2
    }
    x as i16
}
```

**迭代过程示例 (n=144):**
```
初始: x=144, y=72
第1轮: x=72,  y=(72+144/72)/2 = (72+2)/2 = 37
第2轮: x=37,  y=(37+144/37)/2 = (37+3)/2 = 20
第3轮: x=20,  y=(20+144/20)/2 = (20+7)/2 = 13
第4轮: x=13,  y=(13+144/13)/2 = (13+11)/2 = 12
第5轮: x=12,  y=(12+144/12)/2 = (12+12)/2 = 12
收敛: √144 = 12 ✓
```

### 五、绘制填充圆（中点圆算法）

```rust
pub fn fb_fill_circle(fb: &mut [u8], cx: i16, cy: i16, radius: i16, color: u16) {
    let hi = (color >> 8) as u8;
    let lo = (color & 0xFF) as u8;
    let w = WIDTH as usize;
    let mut x = radius;
    let mut y: i16 = 0;
    let mut err: i16 = 1 - radius;

    while x >= y {
        // 利用 8 对称性，画 4 条水平扫描线
        draw_hline(fb, cx - x, cy + y, cx + x, hi, lo, w);
        draw_hline(fb, cx - x, cy - y, cx + x, hi, lo, w);
        draw_hline(fb, cx - y, cy + x, cx + y, hi, lo, w);
        draw_hline(fb, cx - y, cy - x, cx + y, hi, lo, w);

        y += 1;
        if err < 0 {
            err += 2 * y + 1;
        } else {
            x -= 1;
            err += 2 * (y - x) + 1;
        }
    }
}
```

**8 对称性原理：**
```
        cy-x cy+x
          │   │
     ┌────┼───┼────┐
     │    │ ● │    │  圆心 (cx, cy)
     └────┼───┼────┘
          │   │
        cy-x cy+x

圆上一点 (cx+x, cy+y) 对称出 8 个点:
  (cx±x, cy±y) 和 (cx±y, cy±x)

填充时只需画水平扫描线，从左边界到右边界。
```

```rust

    round += 1;
    // 1. 初始化 5 个小球（随机位置）
    let mut balls: [Ball; BALL_COUNT] = core::array::from_fn(|i| {

            for j in (i + 1)..BALL_COUNT { /* ... */ }
        }

        // 4. 更新物理状态
        for ball in balls.iter_mut() { /* 重力 + 地面碰撞 */ }

        // 5. 绘制
        for ball in balls.iter() {
            fb_fill_circle(fb, ball.x, ball.y, BALL_RADIUS, ball.color);

        // 6. 刷新到屏幕
        display.flush(fb);

            display.delay.delay_millis(1000);
            // 渐变清屏效果

                // 逐行擦除...
                if y % 4 == 3 { display.flush(fb); }
            }
            break;
        }
    }
}
```

### 七、伪随机数生成器

嵌入式环境没有操作系统提供的随机数，我们用 xorshift 算法自己实现：

```rust
struct Rng(u32);

impl Rng {
    fn next(&mut self) -> u32 {
        let mut x = self.0;
        x ^= x << 13;
        x ^= x >> 17;
        x ^= x << 5;
        self.0 = x;
        x
    }

    fn range(&mut self, n: u32) -> u32 {
        self.next() % n  // 取模得到 [0, n) 范围
    }
}

// 使用种子 12345 初始化
let mut rng = Rng(12345);
```

**xorshift 特点：**
- 周期 2³²-1（约 42 亿）
- 计算极快（3 次异或 + 3 次移位）
- 分布均匀，适合嵌入式
