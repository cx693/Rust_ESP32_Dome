### 一、正方体的数据定义

#### 顶点坐标

正方体有 8 个顶点，以原点为中心，边长为 2：

```rust
const VERTS: [[f32; 3]; 8] = [
    [-1.0, -1.0, -1.0],  // 0: 左下后
    [ 1.0, -1.0, -1.0],  // 1: 右下后
    [ 1.0,  1.0, -1.0],  // 2: 右上后
    [-1.0,  1.0, -1.0],  // 3: 左上后
    [-1.0, -1.0,  1.0],  // 4: 左下前
    [ 1.0, -1.0,  1.0],  // 5: 右下前
    [ 1.0,  1.0,  1.0],  // 6: 右上前
    [-1.0,  1.0,  1.0],  // 7: 左上前
];
```

#### 面定义

每个面由 4 个顶点组成（逆时针方向）：

```rust
const FACES: [[usize; 4]; 6] = [
    [4, 5, 6, 7],  // 前面 (z=+1)
    [1, 0, 3, 2],  // 后面 (z=-1)
    [0, 4, 7, 3],  // 左面 (x=-1)
    [5, 1, 2, 6],  // 右面 (x=+1)
    [0, 1, 5, 4],  // 底面 (y=-1)
    [3, 7, 6, 2],  // 顶面 (y=+1)
];
```

#### 面法向量

每个面的法向量垂直于该面，指向面的外侧：

```rust
const FACE_NORMALS: [[f32; 3]; 6] = [
    [ 0.0,  0.0,  1.0],  // 前面: +Z 方向
    [ 0.0,  0.0, -1.0],  // 后面: -Z 方向
    [-1.0,  0.0,  0.0],  // 左面: -X 方向
    [ 1.0,  0.0,  0.0],  // 右面: +X 方向
    [ 0.0, -1.0,  0.0],  // 底面: -Y 方向
    [ 0.0,  1.0,  0.0],  // 顶面: +Y 方向
];
```

### 二、3D 旋转变换

绕 X 轴和 Y 轴旋转的组合变换：

```rust
fn rotate(p: [f32; 3], ax: f32, ay: f32) -> [f32; 3] {
    let (sx, cx) = (sinf(ax), cosf(ax));
    let (sy, cy) = (sinf(ay), cosf(ay));

    // 绕 X 轴旋转
    // y' = y·cos(ax) - z·sin(ax)
    // z' = y·sin(ax) + z·cos(ax)
    let y1 = p[1] * cx - p[2] * sx;
    let z1 = p[1] * sx + p[2] * cx;

    // 绕 Y 轴旋转
    // x' = x·cos(ay) + z'·sin(ay)
    // z'' = -x·sin(ay) + z'·cos(ay)
    let x2 = p[0] * cy + z1 * sy;
    let z2 = -p[0] * sy + z1 * cy;

    [x2, y1, z2]
}
```

**旋转矩阵图解：**
```
绕 X 轴旋转 (ax):          绕 Y 轴旋转 (ay):
    y                           x
    ↑                           ↑
    │  ↻ ax                     │  ↻ ay
    └──→ z                      └──→ z

组合: 先绕 X 转，再绕 Y 转
```

### 三、透视投影

将 3D 坐标投影到 2D 屏幕：

```rust
fn project(p: [f32; 3], dist: f32, scale: f32) -> (i16, i16) {
    let z = p[2] + dist;  // 物体到相机的距离
    if z < 1.0 {
        return (120, 120);  // 避免除以零
    }
    // 透视除法: 屏幕坐标 = 3D坐标 × 焦距 / Z距离
    let x = (p[0] * scale / z) as i16 + 120;  // 屏幕中心 x=120
    let y = (p[1] * scale / z) as i16 + 120;  // 屏幕中心 y=120
    (x, y)
}
```

**透视投影原理：**
```
        相机
          │
          │ dist (相机到投影面的距离)
          │
    ┌─────┼─────┐  ← 投影面 (屏幕)
    │     │     │
    │     ●─────│─── 物体 (x, y, z)
    │     │     │
    └─────┼─────┘
          │
          ↓
    屏幕坐标 = x × scale / (z + dist)
    
    z 越大（越远），投影越小 → 近大远小
```

### 四、背面剔除

只有面向相机的面才需要绘制。通过面法向量与视线方向的点积判断：

```rust
for i in 0..6 {
    // 旋转后的法向量
    if rn[i][2] >= 0.0 {
        continue;  // 法向量 Z 分量 >= 0 → 背面，跳过
    }
    // ... 收集可见面
}
```

**判断原理：**
```
        相机 (看向 -Z 方向)
          │
          ↓ -Z
    ┌─────────────┐
    │ 面 A (法向量 +Z) │  ← rn[2] > 0 → 背面，不绘制
    └─────────────┘
          │
    ┌─────────────┐
    │ 面 B (法向量 -Z) │  ← rn[2] < 0 → 正面，绘制
    └─────────────┘

点积: normal · view_dir = rn[2] × (-1)
  > 0 → 背面（法向量与视线同向）
  < 0 → 正面（法向量与视线反向）
```

### 五、深度排序（画家算法）

多个面重叠时，需要按深度排序，从远到近绘制：

```rust
// 收集可见面及其深度
let mut vis: [(usize, f32); 6] = [(0, 0.0); 6];
let mut cnt = 0usize;
for i in 0..6 {
    if rn[i][2] >= 0.0 { continue; }
    let f = &FACES[i];
    let depth = rv[f[0]][2] + rv[f[1]][2] + rv[f[2]][2] + rv[f[3]][2];
    vis[cnt] = (i, depth);
    cnt += 1;
}

// 选择排序（从远到近）
for i in 0..cnt {
    for j in i + 1..cnt {
        if vis[j].1 > vis[i].1 {
            vis.swap(i, j);
        }
    }
}
```

**画家算法原理：**
```
像画家一样，先画远处的物体，再画近处的物体。
近处的面会覆盖远处的面，自然形成正确的遮挡关系。

远处 (depth 大)          近处 (depth 小)
    ┌───┐                   ┌───┐
    │ A │  ← 先画 A         │ B │  ← 后画 B，覆盖 A
    └───┘                   └───┘
```

### 六、绘制填充三角形

正方体的每个面由 2 个三角形组成：

```rust
// 一个面 = 2 个三角形
fb_fill_triangle(fb, ax, ay, bx, by, cx, cy, color);  // 三角形 1
fb_fill_triangle(fb, ax, ay, cx, cy, dx, dy, color);  // 三角形 2

// 黑色边框线
fb_draw_line(fb, ax, ay, bx, by, COLOR_RGB565_BLACK);
fb_draw_line(fb, bx, by, cx, cy, COLOR_RGB565_BLACK);
fb_draw_line(fb, cx, cy, dx, dy, COLOR_RGB565_BLACK);
fb_draw_line(fb, dx, dy, ax, ay, COLOR_RGB565_BLACK);
```

**三角形拆分：**
```
A ──── B          A ──── B
│     │    →     ╱ ╲
│     │        A╱   ╲B
D ──── C       ╲   ╱
                ╲ ╱
D ──── C
```

### 七、局部刷新优化

每帧只更新正方体所在的区域，而不是全屏刷新：

```rust
// 计算包围盒
fn bbox(pts: &[(i16, i16); 8]) -> (u16, u16, u16, u16) {
    let (mut x0, mut x1) = (pts[0].0, pts[0].0);
    let (mut y0, mut y1) = (pts[0].1, pts[0].1);
    for p in pts.iter().skip(1) {
        if p.0 < x0 { x0 = p.0; }
        if p.0 > x1 { x1 = p.0; }
        if p.1 < y0 { y0 = p.1; }
        if p.1 > y1 { y1 = p.1; }
    }
    (
        (x0 - 3).max(0) as u16,  // 扩展 3 像素边距
        (y0 - 3).max(0) as u16,
        ((x1 + 3).min(239)) as u16,
        ((y1 + 3).min(239)) as u16,
    )
}

// 合并当前帧和上一帧的包围盒
let region = if let Some(prev) = prev_bb {
    (
        prev.0.min(bb.0),  // 取两个包围盒的并集
        prev.1.min(bb.1),
        prev.2.max(bb.2),
        prev.3.max(bb.3),
    )
} else {
    (0, 0, 239, 239)
};

// 只清屏和刷新该区域
fb_fill_rect(fb, region.0, region.1, region.2, region.3, COLOR_RGB565_BLACK);
// ... 绘制 ...
display.flush_region(fb, region.0, region.1, region.2, region.3);
```

**性能对比：**
```
全屏刷新: 240 × 240 × 2 = 115,200 字节/帧
局部刷新: ~100 × 100 × 2 = ~20,000 字节/帧  (正方体约占屏幕 40%)
性能提升: ~5-6 倍
```

### 八、动画循环

```rust
let mut ax: f32 = 0.0;  // X 轴旋转角度
let mut ay: f32 = 0.0;  // Y 轴旋转角度
let mut prev_bb: Option<(u16, u16, u16, u16)> = None;

loop {
    // 1. 旋转所有顶点和法向量
    // 2. 投影到 2D
    // 3. 计算包围盒
    // 4. 清屏（局部区域）
    // 5. 背面剔除 + 深度排序
    // 6. 绘制可见面
    // 7. 局部刷新
    // 8. 更新角度

    ay += 0.04;  // Y 轴旋转速度
    ax += 0.03;  // X 轴旋转速度
    if ay > 6.2832 { ay -= 6.2832; }  // 2π = 一圈
    if ax > 6.2832 { ax -= 6.2832; }
}
```
