本篇是第一篇，目标：**在屏幕上显示中英文混合的抗锯齿文字**。

### 接线图

```
ESP32-S3          ST7789 LCD
─────────────────────────────
GPIO48 (SCK)  ──→ SCK
GPIO47 (MOSI) ──→ SDA/MOSI
GPIO2         ──→ DC  (数据/命令选择)

```

## 📝 核心代码讲解

### 一、SPI 初始化

SPI（Serial Peripheral Interface）是 ESP32 与 LCD 通信的接口。我们需要配置：

```rust
// main.rs — SPI 初始化
let (rx_buf, rx_desc, tx_buf, tx_desc) = dma_buffers!(32000);
let spi = Spi::new(
    peripherals.SPI2,
    SpiConfig::default()
        .with_frequency(Rate::from_khz(80000))  // 80MHz 时钟
        .with_mode(Mode::_3),                     // SPI 模式 3
)
.unwrap()
.with_sck(peripherals.GPIO48)    // 时钟线
.with_mosi(peripherals.GPIO47)   // 数据线
.with_dma(peripherals.DMA_CH0)   // 使用 DMA 通道 0
.with_buffers(
    DmaRxBuf::new(rx_desc, rx_buf).unwrap(),
    DmaTxBuf::new(tx_desc, tx_buf).unwrap(),
);
```

**关键参数说明：**

| 参数 | 值 | 说明 |
|------|-----|------|
| 时钟频率 | 80 MHz | ST7789 最高支持约 100MHz |
| SPI 模式 | Mode 3 | CPOL=1, CPHA=1（空闲高电平，下降沿采样） |
| DMA 缓冲区 | 32000 字节 | 单次 DMA 传输最大容量 |

#### DMA vs 阻塞传输

```
阻塞传输 (Spi):
  CPU → 写入 SPI 寄存器 → 等待完成 → 返回
  问题: CPU 在等待期间什么也干不了

DMA 传输 (SpiDmaBus):
  CPU → 配置 DMA 描述符 → DMA 自动搬运数据 → CPU 继续做其他事
  优势: CPU 可以并行处理其他任务
```

本项目使用 DMA 传输，**刷新速度提升约 3-5 倍**。

### 二、ST7789 初始化

ST7789 是一颗 LCD 控制器芯片，需要通过 SPI 发送一系列命令来配置：

```rust
// st7789.rs — 初始化流程
pub fn init(&mut self) {
    // 1. 开背光
    if let Some(ref mut blk) = self.blk {
        blk.set_high();
    }

    // 2. 硬件复位
    self.hard_reset();  // RES 拉低 20ms → 拉高等待 150ms

    // 3. 软件复位
    self.write_cmd(0x01);  // SWRESET
    self.delay.delay_millis(200);

    // 4. 退出休眠
    self.write_cmd(0x11);  // SLPOUT
    self.delay.delay_millis(120);

    // 5. 设置像素格式: RGB565 (16bit/pixel)
    self.write_cmd(0x3A);  // COLMOD
    self.write_data(&[0x55]);

    // 6. 电气参数配置（伽马校正、电压等）
    // ... 省略详细参数 ...

    // 7. 颜色反转（某些屏幕需要）
    self.write_cmd(0x21);  // INVON

    // 8. 开启显示
    self.write_cmd(0x29);  // DISPON
}
```

**RGB565 像素格式：**
```
  RRRRRGGGGGGBBBBB
  15    10 9    5 4     0
  ├─────┤├──────┤├──────┤
   5bit    6bit    5bit
  
  红色: 0xF800 = 11111_000000_00000
  绿色: 0x07E0 = 00000_111111_00000
  蓝色: 0x001F = 00000_000000_11111
```

### 三、帧缓冲区（Framebuffer）

帧缓冲区是一块内存区域，每个像素占 2 字节（RGB565），按行排列：

```
帧缓冲区布局 (240×240):
┌─────────────────────────────────┐
│ [0,0] [1,0] [2,0] ... [239,0]  │  第 0 行 (480 字节)
│ [0,1] [1,1] [2,1] ... [239,1]  │  第 1 行
│  ...                            │
│ [0,239] ...         [239,239]   │  第 239 行
└─────────────────────────────────┘
总大小: 240 × 240 × 2 = 115,200 字节 ≈ 112.5 KB
```

#### 画点函数

```rust
pub fn fb_set_pixel(fb: &mut [u8], x: i16, y: i16, color: u16) {
    // 边界检查
    if x < 0 || y < 0 || x >= WIDTH as i16 || y >= HEIGHT as i16 {
        return;
    }
    // 计算像素在缓冲区中的偏移
    let idx = ((y as usize) * (WIDTH as usize) + (x as usize)) * 2;
    // 写入高字节和低字节（大端序）
    fb[idx] = (color >> 8) as u8;
    fb[idx + 1] = (color & 0xFF) as u8;
}
```

#### 刷屏函数（DMA 分块传输）

```rust
pub fn flush(&mut self, fb: &[u8]) {
    self.set_window(0, 0, WIDTH - 1, HEIGHT - 1);
    self.dc.set_high();  // 数据模式
    // 按 DMA_MAX_SIZE 分块传输（ESP32 DMA 描述符 15bit 长度限制）
    for chunk in fb.chunks(DMA_MAX_SIZE) {
        self.spi.write(chunk).unwrap();
    }
}
```

### 四、字体渲染系统

这是本项目最核心的部分。我们使用 **编译期光栅化** 方案：

#### 原理

```
传统方案（运行时渲染）:
  TTF 字体文件 → 解析字形 → 光栅化 → 显示
  问题: 需要浮点运算 + 大量内存，嵌入式设备吃不消

本方案（编译期渲染）:
  build.rs 在编译时:
    1. 扫描源码中所有 draw_str() 调用
    2. 提取文本内容、字体、字号
    3. 用 ab_glyph 库光栅化每个字符
    4. 生成 Rust 代码嵌入字模数据
  
  运行时只需:
    查表 → 写入帧缓冲区
```

#### build.rs 核心逻辑

```rust
// build.rs — 编译期字体光栅化
```

#### 抗锯齿渲染

```rust
// font.rs — 像素混合
pub fn blend_rgb565(fg: u16, bg: u16, alpha: u8) -> u16 {
    if alpha == 0 { return bg; }       // 完全透明
    if alpha == 255 { return fg; }     // 完全不透明
    
    let a = alpha as u32;
    let inv = 255 - a;
    // 分离 RGB565 的 R/G/B 分量
    let fr = ((fg >> 11) & 0x1F) as u32;
    let fg_g = ((fg >> 5) & 0x3F) as u32;
    let fb = (fg & 0x1F) as u32;
    let br = ((bg >> 11) & 0x1F) as u32;
    let bg_g = ((bg >> 5) & 0x3F) as u32;
    let bb = (bg & 0x1F) as u32;
    // Alpha 混合: result = fg * alpha + bg * (1 - alpha)
    let r = (fr * a + br * inv) / 255;
    let g = (fg_g * a + bg_g * inv) / 255;
    let b = (fb * a + bb * inv) / 255;
    ((r << 11) | (g << 5) | b) as u16
}
```

**效果对比：**
```
无抗锯齿:          有抗锯齿:
  ██████             ░▓████
  ██████             ░▓████
  ██████             ░▓████
  边缘锯齿明显       边缘平滑过渡
```

### 五、主程序

```rust
// main.rs
#[main]
fn main() -> ! {
    // 1. 初始化外设
    let peripherals = esp_hal::init(esp_hal::Config::default());

    // 2. 配置 GPIO
    let dc = Output::new(peripherals.GPIO2, Level::Low, OutputConfig::default());
    let res = Output::new(peripherals.GPIO1, Level::High, OutputConfig::default());
    let blk = Output::new(peripherals.GPIO0, Level::High, OutputConfig::default());

    // 3. 初始化 SPI + DMA
    // ... (见上文)

    // 4. 初始化 LCD
    let mut display = St7789::new(spi, dc, res).with_blk(blk);
    display.init();

    // 5. 清屏
    let fb = unsafe { &mut *FB.0.get() };
    fb_clear(fb, BLACK);

    // 6. 绘制文字
    let mut y = 4i16;
    draw_str(fb, 0, y, "Hello world", FontFamily::Cxi, 16, GREEN, BLACK, W, H);
    y += 20;
    draw_str(fb, 0, y, "你好世界", FontFamily::Cxi, 20, YELLOW, BLACK, W, H);
    y += 26;
    draw_str(fb, 0, y, "Hello world", FontFamily::AliMaMa, 24, WHITE, BLACK, W, H);
    y += 30;
    draw_str(fb, 0, y, "你好世界!", FontFamily::AliMaMa, 24, CYAN, BLACK, W, H);

    // 7. 刷新到屏幕
    display.flush(fb);

    loop {}  // 嵌入式程序永不返回
}
```

## 🔍 关键知识点总结

| 知识点 | 说明 |
|--------|------|
| SPI 模式 3 | CPOL=1, CPHA=1，ST7789 要求的通信模式 |
| DMA 传输 | CPU 配置后由硬件自动搬运数据，不阻塞 CPU |
| RGB565 | 16bit 像素格式，5R+6G+5B，嵌入式常用 |
| 帧缓冲区 | 先在内存中绘制，再一次性刷新到屏幕 |
| 编译期光栅化 | build.rs 在编译时将字体渲染为字模数据 |
| Alpha 混合 | 用覆盖率实现字体抗锯齿，边缘更平滑 |
