## 📝 核心代码讲解

### 一、GIF 文件格式

GIF 文件由以下部分组成：

```
┌─────────────────────────────────────────┐
│            GIF Header (6 bytes)         │  "GIF89a" 或 "GIF87a"
├─────────────────────────────────────────┤
│      Logical Screen Descriptor (7B)     │  宽、高、全局颜色表标志
├─────────────────────────────────────────┤
│      Global Color Table (可选)          │  256 色 × 3 字节 = 768 字节
├─────────────────────────────────────────┤
│      Extension Block (可选)             │  图形控制扩展、注释等
├─────────────────────────────────────────┤
│      Image Descriptor                   │  帧位置、尺寸、局部颜色表
├─────────────────────────────────────────┤
│      Image Data                         │  LZW 压缩的像素数据
├─────────────────────────────────────────┤
│      ... (重复多帧)                      │
├─────────────────────────────────────────┤
│      Trailer (0x3B)                     │  文件结束标志
└─────────────────────────────────────────┘
```

### 二、GIF 解析器

```rust
pub struct GifPlayer<'a> {
    data: &'a [u8],        // GIF 文件数据（编译期嵌入）
    pos: usize,            // 当前读取位置
    global_ct: [u8; 768],  // 全局颜色表 (256色 × RGB)
    frame_count: usize,    // 总帧数
    data_start: usize,     // 第一帧数据的起始位置
    // 帧间状态
    prev_disposal: u8,     // 上一帧的处置方法
    prev_left: i16,        // 上一帧的位置
    prev_top: i16,
    prev_width: usize,
    prev_height: usize,
}
```

#### 初始化流程

```rust
fn init(&mut self) {
    // 1. 验证 GIF 魔数
    if &self.data[0..6] != b"GIF89a" && &self.data[0..6] != b"GIF87a" {
        return;
    }

    // 2. 解析逻辑屏幕描述符
    let width = u16::from_le_bytes([data[6], data[7]]);
    let height = u16::from_le_bytes([data[8], data[9]]);
    let flags = data[10];

    // 3. 读取全局颜色表
    let has_gct = flags & 0x80 != 0;
    if has_gct {
        let gct_size = 1 << ((flags & 0x07) + 1);  // 2^(N+1) 色
        self.global_ct[..gct_size * 3]
            .copy_from_slice(&data[13..13 + gct_size * 3]);
    }

    // 4. 统计帧数
    self.frame_count = self.count_frames();
}
```

### 三、帧解码

```rust
pub fn decode_next_frame(&mut self, fb: &mut [u8]) -> Option<u16> {
    // 处理上一帧的处置方法
    if self.prev_disposal == 2 {
        // 恢复为背景色（黑色）
        fb_fill_rect(fb, prev_left, prev_top, ...);
    }

    loop {
        match self.data[self.pos] {
            0x2C => {
                // 图像描述符 → 解析帧数据
                let left = read_u16_le(data, pos);
                let top = read_u16_le(data, pos + 2);
                let w = read_u16_le(data, pos + 4);
                let h = read_u16_le(data, pos + 6);

                // 读取颜色表（局部或全局）
                let ct = if has_lct { &local_ct } else { &self.global_ct };

                // LZW 解码
                let pixel_count = w * h;
                let mut indexed_pixels = vec![0u8; pixel_count];
                let decoded = Self::decode_lzw(&compressed, min_code_size, ...);

                // 像素索引 → RGB565 → 写入帧缓冲区
                for y in 0..h {
                    for x in 0..w {
                        let color_idx = indexed_pixels[y * w + x];
                        if transparent && color_idx == transparent_index {
                            continue;  // 跳过透明像素
                        }
                        let r = ct[color_idx * 3];
                        let g = ct[color_idx * 3 + 1];
                        let b = ct[color_idx * 3 + 2];
                        let color = rgb_to_rgb565(r, g, b);
                        fb_set_pixel(fb, left + x, top + y, color);
                    }
                }

                return Some(delay_ms);  // 返回帧延迟
            }
            0x21 => {
                // 扩展块（图形控制、注释等）
                // 提取帧延迟、透明色、处置方法
            }
            0x3B => return None,  // 文件结束
        }
    }
}
```

### 四、LZW 解码算法

LZW（Lempel-Ziv-Welch）是 GIF 使用的无损压缩算法：

```rust
fn decode_lzw(
    data: &[u8],           // 压缩数据
    min_code_size: usize,  // 最小码字长度
    pixels: &mut [u8],     // 输出像素索引
    pixel_count: usize,    // 像素总数
) -> usize {
    let clear_code = 1 << min_code_size;  // 清除码
    let eoi_code = clear_code + 1;        // 结束码

    // LZW 字典（静态数组，避免堆分配）
    static mut PARENTS: [u16; 4096] = [0; 4096];     // 父节点
    static mut FIRST_BYTE: [u8; 4096] = [0u8; 4096]; // 首字节
    static mut LENGTHS: [u16; 4096] = [0u16; 4096];  // 序列长度
    static mut STACK: [u8; 4096] = [0u8; 4096];      // 输出栈

    let mut code_size = min_code_size + 1;  // 当前码字长度
    let mut next_code = eoi_code + 1;       // 下一个可用码字
    let mut bit_buf: u32 = 0;               // 位缓冲区
    let mut bits_in_buf: usize = 0;
    let mut prev_code: Option<usize> = None;

    loop {
        // 读取一个码字
        while bits_in_buf < code_size {
            bit_buf |= (data[byte_pos] as u32) << bits_in_buf;
            bits_in_buf += 8;
            byte_pos += 1;
        }
        let code = (bit_buf & ((1u32 << code_size) - 1)) as usize;
        bit_buf >>= code_size;
        bits_in_buf -= code_size;

        match code {
            eoi_code => break,          // 结束
            clear_code => {             // 重置字典
                code_size = min_code_size + 1;
                next_code = eoi_code + 1;
                prev_code = None;
                continue;
            }
            _ => {
                // 解码码字 → 输出像素序列
                // ... (核心 LZW 逻辑)

                // 将新序列加入字典
                if next_code < 4096 {
                    PARENTS[next_code] = prev as u16;
                    FIRST_BYTE[next_code] = first_byte;
                    LENGTHS[next_code] = LENGTHS[prev] + 1;
                    next_code += 1;
                    if next_code >= (1 << code_size) && code_size < 12 {
                        code_size += 1;  // 码字长度增加
                    }
                }
            }
        }
    }
}
```

**LZW 算法核心思想：**
```
初始字典: [0], [1], [2], ... [255], [CLEAR], [EOI]
          ← 码字 0-255 对应单个像素值

编码过程:
  输入: A B A B A B A B
  字典扩展:
    256: [A B]
    257: [B A]
    258: [A B A]
    259: [B A B]
  输出: 65 66 256 258  (A B [AB] [ABA])

解码过程:
  读取 65 → 输出 A
  读取 66 → 输出 B, 字典添加 256: [A B]
  读取 256 → 输出 A B, 字典添加 257: [B A]
  ...
```

### 五、帧处置方法

GIF 支持多种帧间过渡方式：

```rust
// 处置方法 (Disposal Method)
match disposal_method {
    0 => { /* 不处置，保持当前帧 */ }
    1 => { /* 不处置，不恢复背景 */ }
    2 => {
        // 恢复为背景色
        // 下一帧解码前，先清除当前帧区域
    }
    3 => {
        // 恢复为前一帧
        // 需要保存前一帧的状态
    }
}
```

**实际效果：**
```
处置方法 0: 帧叠加（最常用）
  帧1: ●      帧2: ●●     帧3: ●●●
  
处置方法 2: 清除后绘制
  帧1: ●      帧2:  ●     帧3:   ●
```

fn main() -> ! {
    // 1. 初始化堆分配器（GIF 解码需要 Vec）
    esp_alloc::heap_allocator!(size: 96 * 1024);

    // 2. 初始化 LCD (DMA)
    // ...

    // 3. 嵌入 GIF 文件（编译期）
    let gif_data = include_bytes!("../../img/J1.gif");
    let mut player = GifPlayer::new(gif_data);
    rprintln!("GIF has {} frames", player.frame_count());

    // 4. 循环播放
    loop {
        player.reset();
        loop {
            match player.decode_next_frame(fb) {
                Some(delay_ms) => {
                    display.flush(fb);  // 刷新到屏幕
                    if delay_ms > 0 {
                        delay.delay_millis(delay_ms as u32);  // 等待帧延迟
                    }
                }
                None => break,  // 一帧播放完毕
            }
        }
    }
}

### 七、RGB565 转换

```rust
fn rgb_to_rgb565(r: u8, g: u8, b: u8) -> u16 {
    let r5 = (r >> 3) as u16;  // 8bit → 5bit
    let g6 = (g >> 2) as u16;  // 8bit → 6bit
    let b5 = (b >> 3) as u16;  // 8bit → 5bit
    (r5 << 11) | (g6 << 5) | b5
}
```

**颜色精度损失：**
```
原始 RGB:  R=0xFF (255), G=0xFF (255), B=0xFF (255)
           ↓ 截断
RGB565:    R=0x1F (31),  G=0x3F (63),  B=0x1F (31)
           ↓ 还原
RGB888:    R=0xF8 (248), G=0xFC (252), B=0xF8 (248)
           
误差: R=7, G=3, B=7  ← 人眼几乎看不出
```

## 🔧 嵌入式内存管理

GIF 解码需要动态内存（Vec），ESP32-S3R8N8 有 8MB PSRAM：

```rust
// Cargo.toml
[dependencies]
esp-alloc = "0.10.0"

// main.rs
esp_alloc::heap_allocator!(size: 96 * 1024);  // 分配 96KB 堆
```

**内存使用估算：**
```
帧缓冲区: 240 × 240 × 2 = 115,200 字节 ≈ 112 KB
GIF 解码临时: ~50 KB (Vec + LZW 字典)
颜色表: 768 字节
总计: ~163 KB
```
