# ESP-HAL Wi-Fi 示例模式分析文档

## 概述
本文档基于 `esp-rs/esp-hal` 官方仓库中 `examples/wifi` 目录的内容，对目前官方支持的 Wi-Fi 工作模式及功能示例进行了系统性分析。

目前，官方提供的 Wi-Fi 示例全面基于 Embassy 异步框架（从 `embassy_` 前缀可以看出），这是官方推荐的现代 Wi-Fi 开发方式，能够更好地处理后台任务调度和电源管理。

## 核心支持模式详解
### Station (STA) 模式 / 客户端模式
设备作为终端（客户端）连接到现有的 Wi-Fi 路由器（如家用或企业 Wi-Fi 网络）。这是物联网设备最常见的联网方式。
- `embassy_dhcp`：演示设备连接到 Wi-Fi 后，作为 DHCP 客户端自动获取 IP 地址、网关和子网掩码的完整流程。
- `embassy_sntp`：演示设备连接 Wi-Fi 获取网络后，使用 SNTP（简单网络时间协议）从网络时间服务器同步系统时间。

### Access Point (AP) 模式 / 热点模式
设备自身作为 Wi-Fi 路由器（热点），允许手机、电脑等其他终端设备连接。
- `embassy_access_point`：演示如何启动一个 Wi-Fi 热点，配置 SSID（网络名称）和密码，并处理客户端的连接和断开事件。

### AP + STA 共存模式 (Coexistence)
设备同时开启热点（AP）和客户端（STA）功能。这种模式常用于**设备配网**场景（手机先连上设备的 AP 传递路由器密码，设备再用 STA 连上路由器）或作为 Wi-Fi 中继器。
- `embassy_access_point_with_sta`：明确演示 AP 和 STA 接口同时初始化并运行的场景。
- `embassy_coex`：演示射频共存机制。在 ESP32 系列中，coex 不仅包含 Wi-Fi AP+STA 的共存，通常也包含 Wi-Fi 与 Bluetooth (BLE) 的射频共存调度，确保两者同时工作时不会互相产生严重的射频干扰。

### Sniffer 模式 / 监听模式 (Monitor Mode)
设备不连接到任何特定的 Wi-Fi 网络，而是将网卡设置为混杂模式（Promiscuous Mode），监听并捕获空中所有的 802.11 无线数据包。
- `sniffer`：演示如何抓取空中的 Wi-Fi 管理帧、控制帧或数据帧。此模式常用于无线网络分析、信道扫描、设备发现或安全审计。

### 原始 802.11 帧发送 (Raw TX)
绕过常规的 Wi-Fi 协议栈，直接在底层构造并发送原始的 802.11 MAC 层数据帧。
- `80211_tx`：演示如何手动拼装 802.11 帧（例如构造特定的 Beacon 帧、发送 Deauth 断开连接帧等）。主要用于底层协议研究、自定义 Mesh 网络路由或安全渗透测试。

## 总结与开发建议
1. 常规物联网开发：如果只需让设备连接互联网传输数据，直接参考 `embassy_dhcp` (STA 模式)。
2. 设备配网与中继：如果需要设备既能被手机连接配置，又能连接外部路由器，参考 `embassy_access_point_with_sta`。
3. 拥抱异步框架：官方 Wi-Fi 示例已全面转向 Embassy 异步框架。如果仍在使用同步阻塞的方式编写 Wi-Fi 代码，强烈建议参考这些 `embassy_` 开头的示例进行重构。异步框架能完美解决 Wi-Fi 后台任务调度问题，并能配合 `esp-wifi` 实现 CPU 的自动休眠（低功耗）。
