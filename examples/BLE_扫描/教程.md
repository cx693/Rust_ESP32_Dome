## 一、项目简介

本项目使用 **Rust** 语言在 **ESP32-S3R8N8** 上实现一个 BLE（蓝牙低功耗）扫描器。它能：

- 扫描附近的 BLE 设备广播包
- 解析并打印设备名称、地址、信号强度（RSSI）
- 区分不同广播类型（ADV_IND / SCAN_RSP 等）
- 自动去重，新设备首次出现时才打印

适合 Rust 嵌入式初学者学习 **异步编程、BLE 协议栈、外设驱动** 的基本用法。

| ESP32-S3 引脚 | 调试器引脚 | 说明 |
|---------------|-----------|------|

## 六、代码详解

### 6.1 程序入口

```rust
#[esp_rtos::main]
async fn main(_s: Spawner) {
    // 1. 初始化 RTT 调试输出
    rtt_init_print!();

    // 2. 初始化 ESP32 外设
    let peripherals = esp_hal::init(esp_hal::Config::default());

    // 3. 分配 72KB 堆内存（BLE 协议栈需要较大堆空间）
    esp_alloc::heap_allocator!(size: 72 * 1024);

    // 4. 启动 Embassy 异步运行时
    let sw_int = SoftwareInterruptControl::new(peripherals.SW_INTERRUPT);
    let timg0 = TimerGroup::new(peripherals.TIMG0);
    esp_rtos::start(timg0.timer0, sw_int.software_interrupt0);

    // 5. 创建 BLE 控制器
    let connector = BleConnector::new(peripherals.BT, Default::default()).unwrap();
    let controller: ExternalController<_, 1> = ExternalController::new(connector);

    // 6. 运行扫描
    ble_scanner_run(controller).await;
}
```

**初始化流程图：**

```
RTT 初始化 → HAL 外设初始化 → 堆分配 → Embassy 启动 → BLE 控制器创建 → 开始扫描
```

> **`#[esp_rtos::main]`** 宏做了什么？
> - 设置中断向量表
> - 初始化 RTOS 调度器
> - 将 `main` 函数包装为 Embassy 任务
> - 处理 `Spawner`（任务调度器）

### 6.2 BLE 扫描核心逻辑

```rust
async fn ble_scanner_run<C>(controller: C)
where
    C: Controller + ControllerCmdSync<LeSetScanParams>,
{
    // 设置本机随机地址（防止被追踪）
    let address = Address::random([0xff, 0x8f, 0x1b, 0x05, 0xe4, 0xff]);

    // 构建协议栈
    let mut resources = HostResources::new();
    let stack = trouble_host::new(controller, &mut resources)
        .set_random_address(address);

    // 解构出 Central 角色（扫描者）和事件循环 Runner
    let Host { central, mut runner, .. } = stack.build();

    // 并发运行：事件循环 + 扫描任务
    join(
        runner.run_with_handler(&printer),  // 收发 HCI 数据包
        async {
            let config = ScanConfig {
                active: true,                    // 主动扫描
                phys: PhySet::M1,                // 1M PHY
                interval: Duration::from_secs(1),
                window: Duration::from_secs(1),
                ..ScanConfig::default()
            };
            let _session = scanner.scan(&config).await.unwrap();
            loop { Timer::after(Duration::from_secs(1)).await; }
        }
    ).await;
}
```

**BLE 扫描工作原理：**

```
┌─────────────┐     广播包      ┌──────────────┐
│  BLE 设备    │ ──────────────→│ ESP32-S3     │
│ （被扫描方）  │               │ （扫描方）     │
└─────────────┘                └──────┬───────┘
                                      │
                               on_adv_reports()
                                      │
                                      ▼
                              ┌──────────────┐
                              │  Printer     │
                              │  去重 + 打印  │
                              └──────────────┘
```

> **主动扫描 vs 被动扫描：**
> - **被动扫描（Passive）**：只监听广播包
> - **主动扫描（Active）**：收到 ADV 后发送 SCAN_REQ，设备会回复 SCAN_RSP（可携带设备名）

### 6.3 广播数据解析

BLE 广播数据是一串 TLV（Type-Length-Value）格式：

```
[长度][AD Type][AD Data...][长度][AD Type][AD Data...]...
```

```rust
fn parse_device_name(data: &[u8]) -> Option<&str> {
    let mut i = 0;
    while i < data.len() {
        let len = data[i] as usize;        // 当前 AD 结构的长度
        if len == 0 || i + len > data.len() {
            break;                          // 数据结束或损坏
        }
        let ad_type = data[i + 1];          // AD 类型
        let ad_data = &data[i + 2..i + 1 + len];  // AD 数据

        // 0x09 = 完整设备名, 0x08 = 缩短设备名
        if matches!(ad_type, 0x09 | 0x08) {
            if let Ok(name) = core::str::from_utf8(ad_data) {
                return Some(name);
            }
        }
        i += 1 + len;  // 跳到下一个 AD 结构
    }
    None
}
```

**常见 AD Type：**

| AD Type | 含义 | 说明 |
|---------|------|------|
| `0x01` | Flags | 广播模式标志 |
| `0x08` | Shortened Local Name | 缩短的设备名 |
| `0x09` | Complete Local Name | 完整设备名 |
| `0x0A` | TX Power Level | 发射功率 |
| `0xFF` | Manufacturer Specific | 厂商自定义数据 |

### 6.4 事件处理与去重

```rust
struct Printer {
    seen: RefCell<Deque<BdAddr, 128>>,  // 环形缓冲区，最多记录 128 个设备
}

impl EventHandler for Printer {
    fn on_adv_reports(&self, mut it: LeAdvReportsIter<'_>) {
        let mut seen = self.seen.borrow_mut();
        while let Some(Ok(report)) = it.next() {
            let is_new = !seen.iter().any(|b| b.raw() == report.addr.raw());

            if is_new {
                // 新设备 → 打印完整信息
                rprintln!("[{}] {:?} [{}] rssi:{}", ...);
                if seen.is_full() { seen.pop_front(); }  // 满了就淘汰最旧的
                let _ = seen.push_back(report.addr);
            } else if matches!(kind, LeAdvEventKind::ScanRsp) {
                // 已知设备的 SCAN_RSP → 补充打印设备名
                rprintln!("[SCAN_RSP] {:?} [{}]", ...);
            }
        }
    }
}
```

**去重策略：**

```
设备首次出现 → 打印 → 加入 seen 列表
设备再次出现 → 跳过（除非是 SCAN_RSP 补充名称）
seen 列表满  → 淘汰最旧记录（FIFO 环形缓冲区）
```

> **为什么用 `RefCell<Deque>` 而不是 `Mutex`？**
> 在 `no_std` + 单核环境下，`RefCell` 提供运行时借用检查，比 `Mutex` 更轻量。trouble-host 的 `EventHandler` trait 要求 `&self`（不可变引用），所以内部需要 `RefCell` 实现内部可变性。

## 十、常见问题

### Q1: 编译报错 `target not found`

**原因：** 未安装 Xtensa 工具链
**解决：**

```bash
espup install
source ~/export-esp.sh
```

### Q2: 烧录失败 `Unable to connect to target`

**原因：** 设备未连接或驱动问题
**解决：**

1. 检查 USB 线是否支持数据传输（非仅充电线）
2. 检查设备管理器是否有未识别设备
3. macOS 用户可能需要安装 [CP210x 驱动](https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers)

### Q3: RTT 无输出

**原因：** probe-rs 未正确连接 RTT 通道
**解决：**

```bash
# 单独附加 RTT 监听
probe-rs attach --chip esp32s3
```

### Q4: 扫描不到任何设备

**可能原因：**

1. 附近没有 BLE 设备（手机蓝牙是否开启？）
2. 堆内存不足 → 尝试增大 `heap_allocator!(size: 128 * 1024)`
3. 天线问题 → 嘉立创开发板天线区域是否被遮挡
