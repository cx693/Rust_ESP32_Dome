## 1. 项目简介

本教程将带你用 **Rust 语言** 驱动一块 **ST7789 SPI LCD 屏幕** (240×240), 在 ESP32-S3 上实现一个**彩色雪花飘落 + 积雪**的动画效果。

你将学到:
- ESP32-S3 的 SPI 外设配置
- DMA 传输的基本概念
- ST7789 屏幕初始化流程
- 帧缓冲区 (Framebuffer) 的使用
- 位图 (Bitmap) 碰撞检测
- Rust `no_std` 嵌入式开发模式

**最终效果:** 100 片彩色雪花从屏幕顶部飘落, 落到底部或已有积雪上就停住, 积雪逐渐向上堆叠, 触顶后清屏重新开始。

| 像素格式 | RGB565 (16bit) |

### 2.3 接线方式

```
ESP32-S3          ST7789 LCD
─────────────────────────────
GPIO48 (SCK)  ──→ SCK
GPIO47 (MOSI) ──→ SDA/MOSI
GPIO2         ──→ DC  (数据/命令选择)
GPIO1         ──→ RES (复位)
GPIO0         ──→ BLK (背光)
GND           ──→ GND
3.3V          ──→ VCC
```

> **引脚说明:**
> - **DC (Data/Command):** 低电平=命令, 高电平=数据
> - **RES (Reset):** 低电平复位, 正常工作时拉高
> - **BLK (Backlight):** 高电平点亮背光

## 5. 屏幕驱动详解

### 5.1 SPI 通信基础

SPI (Serial Peripheral Interface) 是一种同步串行通信协议:

```
        主设备 (ESP32)          从设备 (ST7789)
        ────────────           ────────────
SCK  ──→ 时钟信号     ──→
MOSI ──→ 数据输出     ──→
DC   ──→ 命令/数据选择 ──→
RES  ──→ 复位         ──→
```

**ST7789 通信协议:**
- DC=**低电平**: 发送的是**命令** (如: 设置地址、复位)
- DC=**高电平**: 发送的是**数据** (如: 像素颜色值)

```rust
// 发送命令
fn write_cmd(&mut self, cmd: u8) {
    self.dc.set_low();           // DC 拉低 → 告诉屏幕"这是命令"
    self.spi.write(&[cmd]).unwrap();
}

// 发送数据
fn write_data(&mut self, data: &[u8]) {
    self.dc.set_high();          // DC 拉高 → 告诉屏幕"这是数据"
    self.spi.write(data).unwrap();
}
```

### 5.2 ST7789 初始化流程

```
背光开启 → 硬复位 → 软复位 → 退出休眠 → 配置参数 → 开显示
```

```rust
pub fn init(&mut self) {
    // 1. 开背光
    if let Some(ref mut blk) = self.blk { blk.set_high(); }

    // 2. 硬复位 (拉低 RES 20ms, 再拉高等待 150ms)
    self.hard_reset();

    // 3. 软复位
    self.write_cmd(0x01); // SWRESET
    self.delay.delay_millis(200);

    // 4. 退出休眠
    self.write_cmd(0x11); // SLPOUT
    self.delay.delay_millis(120);

    // 5. 配置像素格式 = RGB565
    self.write_cmd(0x3A); // COLMOD
    self.write_data(&[0x55]);

    // 6. 配置电气参数 (Porch、伽马校正等)
    // ... (详见 st7789.rs)

    // 7. 开启显示
    self.write_cmd(0x29); // DISPON
    self.delay.delay_millis(120);
}
```

> **为什么要延迟?**
> ST7789 内部需要时间处理每条命令, 数据手册规定了最小等待时间。不等够时间屏幕可能初始化失败。

### 5.3 RGB565 像素格式

```
RGB565: 每像素 16 bit
┌─────────────────────┐
│ RRRRR GGGGGG BBBBB  │
│ 5bit  6bit   5bit   │
└─────────────────────┘
```

```rust
// 颜色常量示例
const COLOR_RED:   u16 = 0xF800; // 11111 000000 00000
const COLOR_GREEN: u16 = 0x07E0; // 00000 111111 00000
const COLOR_BLUE:  u16 = 0x001F; // 00000 000000 11111
const COLOR_WHITE: u16 = 0xFFFF; // 11111 111111 11111
const COLOR_BLACK: u16 = 0x0000; // 00000 000000 00000
```

### 5.4 帧缓冲区 (Framebuffer)

帧缓冲区是一块内存, 每个字节对应屏幕上的一个像素点:

```
内存布局 (240×240, 每像素 2 字节):
┌─────┬─────┬─────┬─────┬─────┬─────┐
│px0  │px1  │px2  │ ... │px239│ ← 第 0 行
│hi lo│hi lo│hi lo│     │hi lo│
├─────┼─────┼─────┼─────┼─────┤
│px240│px241│ ... │     │     │ ← 第 1 行
└─────┴─────┴─────┴─────┴─────┘
... 共 240 行
总大小: 240 × 240 × 2 = 115,200 字节
```

```rust
// 画点: 根据 (x, y) 计算内存偏移
pub fn fb_set_pixel(fb: &mut [u8], x: i16, y: i16, color: u16) {
    let idx = ((y as usize) * 240 + (x as usize)) * 2;
    fb[idx]     = (color >> 8) as u8;  // 高字节
    fb[idx + 1] = (color & 0xFF) as u8; // 低字节
}
```

> **为什么用帧缓冲区?**
> 直接逐像素写 SPI 效率极低 (每次写 2 字节都要一次 SPI 传输)。帧缓冲区先把所有像素画好, 最后一次性通过 DMA 刷到屏幕, 大幅减少 SPI 调用次数。

### 5.5 DMA 传输

DMA (Direct Memory Access) 让数据传输不经过 CPU:

```
无 DMA: CPU 逐字节搬运 → CPU 被占用, 速度慢
有 DMA: DMA 控制器自动搬运 → CPU 空闲做其他事
```

```rust
// SPI + DMA 初始化链
let spi = Spi::new(peripherals.SPI2, SpiConfig::default()...)
    .unwrap()
    .with_sck(peripherals.GPIO48)      // 时钟引脚
    .with_mosi(peripherals.GPIO47)     // 数据引脚
    .with_dma(peripherals.DMA_CH0)     // 绑定 DMA 通道
    .with_buffers(rx_buf, tx_buf);     // 配置 DMA 缓冲区
```

> **`dma_buffers!(32000)` 是什么?**
> 创建 32000 字节的 DMA 缓冲区。ESP32-S3 的 DMA 描述符有 15bit 长度限制 (最大 32767), 所以大数据需要分块传输。

## 6. 示例程序详解

### 6.1 整体架构

```

│           主循环 (loop)              │
│                                     │
│  ┌─────────────────────────────┐    │

│  │  5. 刷新到屏幕               │    │
│  │  6. 触顶检测 → 清屏重来      │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

### 6.2 核心数据结构

```rust
// 雪花
struct Flake {
    x: i16,      // 水平坐标
    y: i16,      // 垂直坐标
    color: u16,  // RGB565 颜色
    speed: u8,   // 下落速度 (像素/帧)
    drift: i8,   // 水平漂移 (-1, 0, +1)
}

// 积雪位图 (每 bit 对应一个像素)
// 240×240 = 57,600 像素 → 1,800 个 u32
static mut SNOW_MAP: [u32; 1800] = [0; 1800];
```

### 6.3 伪随机数生成器

嵌入式环境没有 `std::rand`, 使用 xorshift32 算法:

```rust
struct Rng(u32);

impl Rng {
    fn next(&mut self) -> u32 {
        let mut x = self.0;
        x ^= x << 13;  // 第一次变换
        x ^= x >> 17;  // 第二次变换
        x ^= x << 5;   // 第三次变换
        self.0 = x;
        x
    }
    // 返回 0..n-1 的随机数
    fn range(&mut self, n: u32) -> u32 {
        self.next() % n
    }
}
```

> **为什么不用 LCG?**
> LCG (线性同余法) 的低位周期短, 容易产生规律性图案。Xorshift32 分布更均匀, 视觉效果更好。

### 6.4 积雪位图操作

用 **1 bit** 标记一个像素是否已积雪, 极大节省内存:

```
像素 (x, y) → 位图索引 = y × 240 + x
位图索引 → 字 = 索引 / 32, 位 = 索引 % 32

例: 像素 (100, 50)
  索引 = 50 × 240 + 100 = 12100
  字 = 12100 / 32 = 378
  位 = 12100 % 32 = 4
  → snow_map[378] 的第 4 bit
```

```rust
// 检查某像素是否已积雪
fn is_snow_at(snow_map: &[u32], x: i16, y: i16) -> bool {
    let idx = y as usize * 240 + x as usize;
    (snow_map[idx / 32] >> (idx % 32)) & 1 != 0
}

// 标记某像素为"已积雪"
fn settle_flake(snow_map: &mut [u32], x: i16, y: i16) {
    let idx = y as usize * 240 + x as usize;
    snow_map[idx / 32] |= 1 << (idx % 32);
}
```

### 6.5 碰撞检测

雪花是 3×3 十字形, 检查下方 3 个点是否碰触积雪:

```
    · X ·     ← 雪花当前位置
    · X ·
    · X ·
  ─────────── ← 检查线 (y+2)
  ? ? ?       ← 下方 3 点有积雪?
```

```rust
fn check_settle(snow_map: &[u32], x: i16, y: i16) -> bool {
    is_snow_at(snow_map, x - 1, y + 2)  // 左下
    || is_snow_at(snow_map, x,     y + 2)  // 正下
    || is_snow_at(snow_map, x + 1, y + 2)  // 右下
}
```

### 6.6 主循环代码

```rust
loop {
    // 1. 清屏为黑色
    fb_clear(fb, COLOR_RGB565_BLACK);

    // 2. 从位图重绘积雪 (4 色分层)
    draw_snow_map(fb, snow_map);

    // 3. 更新每片雪花
    for flake in flakes.iter_mut() {
        // 画十字形
        fb_set_pixel(fb, flake.x, flake.y, flake.color);
        // ... (上下左右各一点)

        // 计算新位置
        let ny = flake.y + flake.speed as i16;
        let nx = flake.x + flake.drift as i16;

        // 随机漂移 (模拟风吹)
        if rng.range(15) == 0 {
            flake.drift = match rng.range(3) {
                0 => -1, 1 => 0, _ => 1
            };
        }

        // 碰到积雪? → 停住, 标记到位图
        if check_settle(snow_map, nx, ny) {
            settle_flake(snow_map, flake.x, flake.y);
            *flake = new_flake(&mut rng, false); // 新雪花从顶部飘入
        } else {
            flake.y = ny;
            flake.x = nx;
        }
    }

    // 4. 刷新到屏幕
    display.flush(fb);

    // 5. 触顶? → 清屏, 开始新一轮
    if snow_reached_top(snow_map) {
        // 逐行擦除动画
        break; // 回到外层循环, 重新开始
    }
}
```

## 8. 常见问题

### Q: 编译报错 `unresolved import esp_hal::dma::dma_buffers`

**原因:** `dma_buffers!` 是 `#[macro_export]` 宏, 导出在 crate 根路径。

**修复:**

```rust
// 错误 ❌
use esp_hal::dma::{DmaRxBuf, DmaTxBuf, dma_buffers};

// 正确 ✅
use esp_hal::dma::{DmaRxBuf, DmaTxBuf};
use esp_hal::dma_buffers;
```

### Q: 编译报错 `Spi` 和 `SpiDmaBus` 类型不匹配

**原因:** `Spi::new()` 返回 `Spi`, 必须经过 DMA 才能变成 `SpiDmaBus`。

**修复:** 补全 DMA 链式调用:

```rust
let spi = Spi::new(peripherals.SPI2, config)
    .unwrap()
    .with_sck(pin)
    .with_mosi(pin)
    .with_dma(peripherals.DMA_CH0)       // ← 必须
    .with_buffers(rx_buf, tx_buf);       // ← 必须
```

### Q: 运行后屏幕没反应

**检查清单:**
1. 接线是否正确 (特别是 DC、RES、BLK)
2. SPI 频率是否过高 (先试 40MHz)
3. SPI 模式是否正确 (ST7789 用 Mode 3)
4. 背光引脚是否拉高

### Q: 屏幕显示颜色偏移/花屏

**可能原因:**
- SPI 模式不对 (试 Mode 0 或 Mode 3)
- 像素格式不对 (确认 COLMOD = 0x55)
- 屏幕偏移量需要调整 (修改 X_OFFSET / Y_OFFSET)

### Q: `stack pointer out of range` 栈溢出

**原因:** 帧缓冲区 115,200 字节太大, 栈放不下。

**修复:** 用 `static` 静态分配:

```rust
static mut FB: [u8; FB_SIZE] = [0u8; FB_SIZE];
let fb = unsafe { &mut *(&raw mut FB) };
```

### Q: `creating a mutable reference to mutable static` 编译错误

**原因:** Rust 2024 edition 禁止 `&mut static mut`。

**修复:** 用 `&raw mut` 裸指针:

```rust
let fb = unsafe { &mut *(&raw mut FB) };
```
