## 一、项目简介

呼吸灯是一种常见的人机交互效果——LED 亮度从暗到亮、再从亮到暗，循环往复，如同呼吸一般。

本项目使用 ESP32-S3 内置的 **LEDC（LED PWM 控制器）** 硬件模块，通过 **硬件渐变（Hardware Fade）** 实现呼吸效果，**CPU 不参与逐级调节占空比**，由硬件自动完成，效率极高。

### 你会学到什么

| 知识点 | 说明 |
|---|---|
| `#![no_std]` | Rust 嵌入式裸机开发的基础 |
| `esp-hal` | ESP32 官方硬件抽象层 |
| LEDC 外设 | ESP32 的 PWM 控制器，专为 LED 调光设计 |
| 定时器 + 通道 | PWM 的两个核心概念 |
| 硬件渐变 | `start_duty_fade` —— 硬件自动完成占空比渐变 |

## 二、硬件准备

### 2.1 开发板

**嘉立创 ESP32-S3R8N8 开发板**，核心参数：

| 参数 | 值 |
|---|---|
| 芯片 | ESP32-S3 |
| Flash | 8MB (N8) |
| PSRAM | 8MB (R8) |
| 内置 LED | GPIO48（板载，无需外接） |

> **嘉立创 ESP32-S3R8N8 开发板自带板载 LED，连接在 GPIO48 引脚上，无需额外接线。**

### 2.2 调试器

使用 **probe-rs** 支持的调试器（如 ESP-Prog、J-Link 等），通过 JTAG 接口烧录和调试。如果使用的是嘉立创开发板，它通常自带 USB 转串口和 JTAG 调试功能，只需一根 USB 线连接电脑即可。

## 三、开发环境搭建

### 3.1 安装 Rust 和 Xtensa 工具链

ESP32-S3 使用 Xtensa 架构，需要安装 Espressif 定制的 Rust 工具链：

```bash
# 安装 espup（Espressif 的 Rust 工具链管理器）
cargo install espup

# 安装 Xtensa 工具链
espup install

# 使环境变量生效
source ~/export-esp.sh
```

### 3.2 安装 probe-rs

```bash
# 安装 probe-rs（烧录和调试工具）
cargo install probe-rs --features cli
```

### 3.3 安装依赖

```bash
# 安装 ESP32 相关的编译依赖
cargo install ldproxy
```

## 五、代码详解

> 完整源码见 `src/bin/main.rs`，以下逐段讲解。

### 5.1 文件头与导入

```rust
//! ESP32-S3 LED 呼吸灯示例
//!
//! 原理：利用 LEDC（LED PWM 控制器）硬件实现 PWM 输出，
//!       通过渐变占空比（duty）模拟呼吸效果。
//!
//! 硬件连接：GPIO48 → LED（开发板内置，通常低电平点亮）

#![no_main]   // 告诉编译器：没有标准 main 函数入口
#![no_std]    // 告诉编译器：不使用标准库（嵌入式没有操作系统）
```

**知识点：**
- `#![no_std]`：嵌入式开发必须。标准库（std）依赖操作系统，而 ESP32 裸机运行无 OS。
- `#![no_main]`：嵌入式程序的入口由芯片的启动代码决定，不是 C 语言的 `main()`。

```rust
use esp_hal::{
    gpio::DriveMode,
    ledc::{
        LSGlobalClkSource, Ledc, LowSpeed,
        channel::{self, ChannelIFace},
        timer::{self, TimerIFace},
    },
    main,
    time::Rate,
};
```

**导入结构解析：**

| 模块 | 用途 |
|---|---|
| `ledc::Ledc` | LEDC 控制器实例 |
| `ledc::timer` | PWM 定时器（决定频率和分辨率） |
| `ledc::channel` | PWM 通道（决定输出到哪个 GPIO） |
| `gpio::DriveMode` | GPIO 驱动模式（推挽/开漏） |
| `time::Rate` | 频率单位封装 |

### 5.2 应用描述符

```rust
esp_bootloader_esp_idf::esp_app_desc!();
```

这行宏展开后会生成一个应用描述符结构体，包含版本号、时间戳等信息。IDF 引导加载程序在启动时会读取这个描述符来验证固件合法性。**每个 ESP-IDF 引导加载程序项目都必须有这行。**

### 5.3 主函数入口

```rust
#[main]        // esp-hal 的入口宏（替代标准的 fn main()）
fn main() -> ! {   // -> ! 表示永不返回（嵌入式程序永远运行）
```

- `#[main]`：由 `esp_hal` 提供，负责初始化堆栈、中断向量表等底层设置。
- `-> !`：**永不返回类型**。嵌入式设备没有"退出程序"的概念，主循环永远运行。

### 5.4 调试初始化

```rust
rtt_init_print!();               // 初始化 RTT 调试通道
rprintln!("呼吸灯 硬件 PWM");    // 打印启动信息
```

**RTT（Real-Time Transfer）** 是 Segger 提出的调试协议，通过 JTAG 接口实现零延迟的调试输出，比串口更稳定。`rprintln!` 宏类似 `println!`，但输出到调试器而非串口。

### 5.5 外设初始化

```rust
let peripherals = esp_hal::init(esp_hal::Config::default());
```

`esp_hal::init()` 是所有 ESP32 Rust 项目的起点。它做两件事：
1. **初始化时钟、GPIO 等系统外设**
2. **返回 `Peripherals` 结构体**，包含所有可用外设的所有权

> **Rust 所有权优势：** 每个外设只能被获取一次，编译期防止多个模块同时操作同一外设。

### 5.6 LEDC 控制器配置

```rust
let mut ledc = Ledc::new(peripherals.LEDC);
ledc.set_global_slow_clock(LSGlobalClkSource::APBClk);
```

**LEDC（LED Control）** 是 ESP32 专为 LED 调光设计的 PWM 控制器，具有硬件渐变能力。

```
LEDC 控制器
├── 全局时钟（这里选 APB 80MHz）
├── 定时器 0~3（决定 PWM 频率和分辨率）
└── 通道 0~7（绑定定时器，输出到 GPIO）
```

**APB 时钟**（Advanced Peripheral Bus）是 ESP32 的外设总线时钟，频率 80MHz。

### 5.7 定时器配置

```rust
let mut lstimer0 = ledc.timer::<LowSpeed>(timer::Number::Timer0);
lstimer0
    .configure(timer::config::Config {
        duty: timer::config::Duty::Duty10Bit,   // 10 位分辨率
        clock_source: timer::LSClockSource::APBClk,
        frequency: Rate::from_khz(5),            // 5kHz
    })
    .unwrap();
```

**定时器参数解析：**

| 参数 | 值 | 含义 |
|---|---|---|
| `Duty10Bit` | 10 位 | 占空比范围 0~1023（2^10 - 1） |
| `APBClk` | 80MHz | 时钟源 |
| `5kHz` | 5000Hz | PWM 频率 |

**PWM 频率计算：**

```
PWM 频率 = 时钟源频率 / (2^分辨率位数 × 分频系数)
5kHz = 80MHz / (1024 × 分频系数)
分频系数 ≈ 15.625（硬件自动计算）
```

> **为什么选 5kHz？** 人眼对低于 ~50Hz 的 PWM 会看到闪烁，5kHz 远超此阈值，LED 看起来是平滑的亮度变化。

### 5.8 通道配置

```rust
let mut channel0 = ledc.channel(channel::Number::Channel0, peripherals.GPIO48);
channel0
    .configure(channel::config::Config {
        timer: &lstimer0,              // 绑定定时器
        duty_pct: 0,                   // 初始占空比 0%
        drive_mode: DriveMode::PushPull, // 推挽输出
    })
    .unwrap();
```

**通道是定时器和 GPIO 之间的桥梁：**

```
定时器（产生 PWM 波形）
   ↓
通道（控制占空比，输出到引脚）
   ↓
GPIO48 → LED
```

- `duty_pct: 0`：初始占空比 0%，LED 熄灭
- `PushPull`：推挽输出模式，能主动输出高/低电平

### 5.9 呼吸灯主循环（核心！）

```rust
loop {
    // 渐亮：0% → 100%，耗时 1000ms
    channel0.start_duty_fade(0, 100, 1000).unwrap();
    while channel0.is_duty_fade_running() {} // 阻塞等待

    // 渐暗：100% → 0%，耗时 1000ms
    channel0.start_duty_fade(100, 0, 1000).unwrap();
    while channel0.is_duty_fade_running() {}
}
```

**这是整个项目的核心——硬件渐变！**

`start_duty_fade(start_pct, end_pct, duration_ms)` 参数说明：

| 参数 | 含义 |
|---|---|
| `start_pct` | 起始占空比百分比 |
| `end_pct` | 结束占空比百分比 |
| `duration_ms` | 渐变持续时间（毫秒） |

**为什么用硬件渐变而非软件循环？**

```
❌ 软件方式（CPU 逐级调节）：
for duty in 0..1024 {
    set_duty(duty);    // CPU 每次都要干预
    delay_ms(1);       // 精度受循环速度影响
}

✅ 硬件方式（本项目）：
start_duty_fade(0, 100, 1000);  // CPU 只需这一行
// 硬件自动平滑渐变，CPU 空闲可做其他事
```

硬件渐变由 LEDC 控制器内部的状态机驱动，**不占用 CPU 周期**，渐变精度由硬件定时器保证。

## 八、常见问题

### Q1: 编译报错 `unknown target 'xtensa-esp32s3-none-elf'`

**原因：** 没有安装 Xtensa 工具链。

```bash
espup install
source ~/export-esp.sh
```

### Q2: 烧录时报错 `No debug probe was found`

**原因：** 调试器未连接或驱动问题。

- 检查 USB 线是否连接
- 嘉立创开发板通常需要安装 **USB-JTAG 驱动**
- macOS 下一般免驱，Windows 需安装 Zadig 或官方驱动

### Q3: LED 没有亮

**排查步骤：**
1. 确认烧录成功（终端有 `呼吸灯 硬件 PWM` 输出）
2. 确认开发板型号（嘉立创 ESP32-S3R8N8 板载 LED 在 GPIO48）
3. 检查 LED 是否低电平点亮（可能需要将代码中 `0→100` 改为 `100→0`）

### Q4: 想修改呼吸速度怎么改？

修改 `start_duty_fade` 的第三个参数（毫秒）：

```rust
// 慢呼吸：3 秒渐变
channel0.start_duty_fade(0, 100, 3000).unwrap();

// 快呼吸：0.5 秒渐变
channel0.start_duty_fade(0, 100, 500).unwrap();
```
