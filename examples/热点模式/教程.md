## 一、项目概述

本教程将带你从零开始，使用 **Rust 语言** 在 **嘉立创 ESP32-S3R8N8** 开发板上实现一个完整的 **WiFi 热点 (AP 模式)** 项目。

### 功能清单

| 功能 | 说明 |
|------|------|
| WiFi AP 热点 | 创建名为 `ESP32-AP` 的 WiFi 热点，WPA2 加密 |
| DHCP 服务器 | 自动为连接设备分配 IP 地址（192.168.4.10~50） |
| Web 仪表盘 | 浏览器访问 `192.168.4.1` 查看已连接设备信息 |
| 实时刷新 | 前端每 2 秒轮询 API，动态更新设备列表 |

### 最终效果

手机或电脑连接 `ESP32-AP` 热点后，打开浏览器访问 `192.168.4.1`，即可看到一个深色主题的控制面板，显示所有已连接设备的 MAC 地址、IP 地址、主机名和在线时长。

## 二、硬件准备

### 2.1 嘉立创 ESP32-S3R8N8 开发板

**ESP32-S3R8N8** 是乐鑫推出的高性能 WiFi + BLE 5.0 SoC，嘉立创提供了高性价比的开发板：

- **处理器**： Xtensa 32 位 LX7 双核，最高 240 MHz
- **内存**： 512 KB SRAM + 8 MB PSRAM (OPI)
- **Flash**： 8 MB (Quad SPI)
- **无线**： 2.4 GHz WiFi (802.11 b/g/n) + Bluetooth 5 (LE)
- **外设**： GPIO、SPI、I2C、UART、ADC、USB OTG 等

### 2.2 硬件连接

嘉立创 ESP32-S3R8N8 开发板只需一根 USB Type-C 数据线连接电脑即可开始开发。开发板自带 USB 转串口芯片，支持自动下载和串口调试。

> **注意**： 部分开发板可能需要按住 `BOOT` 按钮再插入 USB 才能进入下载模式。

## 六、配置文件详解

### 6.1 `.cargo/config.toml`

```toml
[target.xtensa-esp32s3-none-elf]
runner = "probe-rs run --chip=esp32s3 --preverify --always-print-stacktrace --no-location"

[build]
rustflags = [
  "-C", "link-arg=-nostartfiles",
  "-Z", "stack-protector=all",
]
target = "xtensa-esp32s3-none-elf"

[unstable]
build-std = ["alloc", "core"]
```

**关键配置解释**：

- **`target = "xtensa-esp32s3-none-elf"`**：指定编译目标为 ESP32-S3 的 Xtensa 架构，`none` 表示无操作系统，`elf` 表示输出 ELF 格式
- **`runner = "probe-rs run ..."`**：配置 `cargo run` 时自动使用 probe-rs 烧录
- **`build-std = ["alloc", "core"]`**：从源码编译 `core` 和 `alloc` 库（因为 Xtensa 是 Tier 3 目标，没有预编译的标准库）
- **`-nostartfiles`**：不使用标准启动文件（嵌入式有自己的启动流程）
- **`-Z stack-protector=all`**：启用栈保护（安全加固）

### 6.2 `rust-toolchain.toml`

```toml
[toolchain]
channel = "esp"
```

指定使用 Espressif 维护的 `esp` 工具链频道，而非官方的 `stable`/`nightly`。

## 七、代码逐行解析

### 7.1 文件头和全局配置

```rust
#![no_main] // 禁用标准 main 入口，由 esp_rtos 接管启动流程
#![no_std]  // 禁用标准库（嵌入式环境没有 OS，不支持 std）

extern crate alloc; // 使用堆分配（esp_alloc 提供）
```

- **`#![no_main]`**：嵌入式程序没有操作系统的 `main` 函数入口，由运行时（`esp_rtos`）负责初始化并调用我们的异步入口函数
- **`#![no_std]`**：禁用 Rust 标准库 `std`，只使用核心库 `core` 和可选的 `alloc`
- **`extern crate alloc`**：虽然禁用了 `std`，但可以通过 `esp_alloc` 提供的堆分配器使用 `alloc` crate 中的 `String`、`Vec` 等类型

### 7.2 导入模块

### 7.3 嵌入固件元信息和 HTML

### 7.4 数据结构定义

```rust
#[derive(Clone, Debug)]
struct Client {
    mac: [u8; 6],                      // MAC 地址
    ip_octet: u8,                      // IP 最后一段
    hostname: heapless::String<32>,    // 客户端主机名（最长 32 字节）
    connected_at: Instant,             // 连接时间戳
}

static CLIENTS: Mutex<RefCell<HVec<Client, 16>>> = Mutex::new(RefCell::new(HVec::new()));
```

**关键设计**：

- 使用 `heapless::String<32>` 而非 `String`，因为嵌入式环境下堆容量有限，固定容量更安全
- `Mutex<RefCell<HVec>>` 是 `no_std` 环境下共享可变状态的标准模式：
  - `Mutex`（来自 `critical_section`）：关中断保护，确保单线程安全
  - `RefCell`：运行时借用检查，允许在 `&` 引用下修改数据
  - `HVec<Client, 16>`：最多存储 16 个客户端

### 7.5 静态内存分配宏

```rust
macro_rules! mk_static {
    ($t:ty, $val:expr) => {{
        static CELL: static_cell::StaticCell<$t> = static_cell::StaticCell::new();
        CELL.uninit().write($val)
    }};
}
```

Embassy 的任务和网络栈需要 `'static` 生命周期。此宏将值放入全局 `StaticCell` 中，确保数据永远不会被释放。

### 7.6 程序主入口

```rust
#[esp_rtos::main]
async fn main(spawner: Spawner) -> ! {
    rtt_init_print!();

    // 1. 初始化硬件
    let peripherals = esp_hal::init(esp_hal::Config::default().with_cpu_clock(CpuClock::max()));
    esp_alloc::heap_allocator!(size: 64 * 1024);

    // 2. 启动 RTOS 调度器
    let timg0 = TimerGroup::new(peripherals.TIMG0);
    let sw_int = SoftwareInterruptControl::new(peripherals.SW_INTERRUPT);
    esp_rtos::start(timg0.timer0, sw_int.software_interrupt0);

    // 3. 创建 WiFi AP
    let (controller, interfaces) = esp_radio::wifi::new(
        peripherals.WIFI,
        ControllerConfig::default().with_initial_config(Config::AccessPoint(
            AccessPointConfig::default()
                .with_ssid("ESP32-AP")
                .with_password(String::from("12345678"))
                .with_auth_method(AuthenticationMethod::Wpa2Personal),
        )),
    ).unwrap();

    // 4. 创建 TCP/IP 网络栈
    let seed = { let r = Rng::new(); (r.random() as u64) << 32 | r.random() as u64 };
    let (stack, runner) = embassy_net::new(
        interfaces.access_point,
        embassy_net::Config::ipv4_static(StaticConfigV4 {
            address: Ipv4Cidr::new(Ipv4Address::new(192, 168, 4, 1), 24),
            gateway: None,
            dns_servers: Default::default(),
        }),
        mk_static!(StackResources<3>, StackResources::<3>::new()),
        seed,
    );

    // 5. 派生三个异步任务
    spawner.spawn(ap_task(controller).unwrap());
    spawner.spawn(net_task(runner).unwrap());
    spawner.spawn(dhcp_task(stack).unwrap());

    stack.wait_config_up().await;
    rprintln!("AP ready: 192.168.4.1");

    // 6. HTTP 服务器主循环
    // ...
}
```

**执行流程**：

1. **硬件初始化**：配置 CPU 时钟为最高频率，分配 64KB 堆内存
2. **RTOS 启动**：使用 TimerGroup0 作为时钟源，启动异步调度器
3. **WiFi AP 创建**：配置热点 SSID、密码和加密方式
4. **网络栈初始化**：设置静态 IP `192.168.4.1/24`，创建 TCP/IP 协议栈
5. **派生异步任务**：分别启动 AP 保活、网络栈驱动、DHCP 服务
6. **HTTP 服务器**：在 80 端口监听并处理 HTTP 请求

### 7.7 DHCP 服务器实现

DHCP（动态主机配置协议）是让设备自动获取 IP 地址的核心协议。本项目实现了一个精简版的 DHCP 服务器：

```rust
#[embassy_executor::task]
async fn dhcp_task(stack: Stack<'static>) {
    let mut socket = UdpSocket::new(stack, &mut rx_meta, &mut rx_buf, &mut tx_meta, &mut tx_buf);
    socket.bind(67).unwrap();  // DHCP 服务器端口

    loop {
        let (n, _) = socket.recv_from(&mut buf).await.unwrap();
        let pkt = &buf[..n];

        // 校验：最小长度 + BOOTREQUEST + DHCP 魔数
        if pkt.len() < 240 || pkt[0] != 1 || pkt[236..240] != [99, 130, 83, 99] { continue; }

        let msg_type = dhcp_opt(pkt, 53).unwrap()[0]; // 1=Discover, 3=Request
        let mac: [u8; 6] = pkt[28..34].try_into().unwrap();

        // Discover → 分配 IP → Offer
        // Request → 确认 IP → ACK
        let ip_octet = match msg_type {
            1 => assign_or_get(mac, hostname),
            3 => register_client(mac, req_octet, hostname),
            _ => continue,
        };

        // 发送 DHCP 响应
        build_dhcp_resp(&mut resp, code, &xid, &mac, &ip);
        socket.send_to(&resp[..len], dest).await.ok();
    }
}
```

**DHCP 交互流程**：

```
客户端                          ESP32 AP (192.168.4.1)
  |                                  |
  |-------- DHCP Discover --------->|  (广播: "有人能给我 IP 吗？")
  |                                  |
  |<------- DHCP Offer -------------|  (回复: "给你 192.168.4.10")
  |                                  |
  |-------- DHCP Request ---------->|  (确认: "我要 192.168.4.10")
  |                                  |
  |<------- DHCP ACK ---------------|  (确认: "OK，租约 3600 秒")
  |                                  |
```

### 7.8 HTTP 服务器和路由

```rust
loop {
    let mut socket = TcpSocket::new(stack, &mut rx_buf, &mut tx_buf);
    socket.set_timeout(Some(Duration::from_secs(10)));
    if socket.accept(80).await.is_err() { continue; }

    let req = read_request(&mut socket).await;
    let (method, path) = parse_request(&req);

    let (status, ct, body) = match (method, path) {
        ("GET", "/")             => ("200 OK", "text/html", HTML.as_bytes()),
        ("GET", "/api/clients")  => {
            let json = clients_json();
            respond(&mut socket, "200 OK", "application/json", json.as_bytes()).await;
            continue;
        }
        _ => ("404 Not Found", "text/plain", b"404"),
    };

    respond(&mut socket, status, ct, body).await;
    socket.close();
}
```

**路由表**：

| 方法 | 路径 | 响应类型 | 说明 |

| GET | `/api/clients` | application/json | 返回已连接设备的 JSON 数据 |

## 十、关键技术点总结

### 10.1 为什么选择 Rust 做嵌入式？

| 对比项 | C/C++ | Rust |
|--------|-------|------|
| 内存安全 | 手动管理，容易出错 | 编译期保证，零成本抽象 |
| 并发安全 | 依赖程序员纪律 | 所有权系统强制保证 |
| 生态 | ESP-IDF 成熟 | esp-hal 生态快速发展 |
| 学习曲线 | 低 | 较高，但值得投入 |
| 性能 | 优秀 | 同等优秀（零成本抽象） |

### 10.2 Embassy 异步模型

Embassy 是嵌入式 Rust 的异步运行时，核心优势：

- **无堆分配**：任务调度器完全在栈上运行
- **低功耗**：空闲时自动进入低功耗模式
- **零开销**：`async/await` 编译为状态机，没有运行时开销
- **类型安全**：编译期检查资源冲突

### 10.3 `no_std` 开发要点

- 不能使用 `std::String`、`std::Vec`，改用 `heapless::String`、`heapless::Vec`
- 不能使用 `println!`，改用 `rprintln!`（RTT 输出）
- 不能使用 `std::thread`，改用 Embassy 的异步任务
- 共享状态使用 `Mutex<RefCell<T>>` 而非 `Arc<Mutex<T>>`

### 10.4 嘉立创 ESP32-S3R8N8 的 Flash 和 PSRAM 配置

嘉立创开发板通常使用 OPI 模式的 PSRAM 和 Quad SPI 的 Flash。如果烧录后无法正常运行，检查 `.cargo/config.toml` 中是否需要添加 Flash/PSRAM 配置：

```toml
[build]
rustflags = [
  "-C", "link-arg=-nostartfiles",
  "-C", "link-arg=-Tlinkall.x",
]
```

## 十一、常见问题排查

### Q1：编译报错 `can't find crate for core`

**原因**：未安装 Xtensa 工具链。
**解决**：运行 `espup install` 并 `source ~/export-esp.sh`。

### Q2：烧录时报错 `No probe was found`

**原因**：开发板未连接或驱动未安装。
**解决**：
1. 检查 USB 线是否支持数据传输（非充电线）
2. macOS 下通常免驱，Windows 需安装 USB 驱动
3. 尝试按住 `BOOT` 按钮再插入 USB

### Q3：WiFi 热点创建成功但无法连接

**原因**：密码长度不足或加密方式不兼容。
**解决**：确保密码至少 8 位，使用 `Wpa2Personal` 加密。

### Q4：浏览器打开 `192.168.4.1` 显示空白

**原因**：HTML 文件未正确嵌入。
**解决**：检查 `web/index.html` 文件是否存在，以及 `include_str!` 路径是否正确。

### Q5：DHCP 分配 IP 失败

**原因**：UDP socket 绑定端口 67 失败。
**解决**：确保 `StackResources` 的并发连接数足够（至少 3）。

## 十二、扩展方向

本项目是一个基础框架，你可以在此基础上扩展更多功能：

1. **GPIO 控制**：通过 Web 页面控制 LED、继电器等外设
2. **传感器数据**：读取温湿度、气压等传感器数据并展示
3. **OTA 升级**：实现空中固件升级功能
4. **WebSocket**：将轮询改为 WebSocket 推送，实时性更好
5. **STA+AP 共存**：同时连接路由器和创建热点，实现网桥功能
6. **蓝牙配网**：通过 BLE 配置 WiFi 账号密码
