## 1. 项目简介

### 1.1 什么是 I2C 扫描器？

I2C 扫描器是一个实用工具，用于**检测 I2C 总线上连接的所有设备**。它通过向每个可能的地址发送数据，根据设备的应答信号来判断设备是否存在。

### 1.2 为什么需要 I2C 扫描器？

- ✅ **快速定位设备**：无需查看 datasheet，直接获取设备地址
- ✅ **排查硬件问题**：检测接线是否正确、设备是否损坏
- ✅ **学习 I2C 协议**：理解地址应答机制的最佳实践

### 1.3 本项目亮点

- 🦀 **纯 Rust 实现**：安全、高效、现代化
- 📊 **可视化输出**：表格形式展示扫描结果
- 🏷️ **设备识别**：自动识别 50+ 种常见 I2C 设备
- 📚 **详细注释**：每行代码都有中文解释

## 2. I2C 协议基础

### 2.1 I2C 简介

I2C（Inter-Integrated Circuit）是一种**两线式串行通信协议**，由 Philips 公司开发。

**特点：**
- 🔌 **两根线**：SDA（数据线）+ SCL（时钟线）
- 📡 **主从架构**：主机控制通信，从机响应
- 🎯 **多设备支持**：一条总线可连接多个设备
- 📍 **地址寻址**：每个设备有唯一地址

### 2.2 I2C 地址格式

```
7 位地址格式：
┌─────────────────────────────────────┐
│  位6  位5  位4  位3  位2  位1  位0  │
└─────────────────────────────────────┘

8 位地址格式（包含读写位）：
┌─────────────────────────────────────┐
│  位6  位5  位4  位3  位2  位1  位0  R/W │
└─────────────────────────────────────┘
                                    ↑
                              0=写, 1=读
```

**地址范围：**
- 7 位：0x08 - 0x77（有效范围）
- 0x00 - 0x07：保留地址
- 0x78 - 0x7F：10 位地址扩展

### 2.3 I2C 通信时序

```
主机发送 START 条件
        ↓
发送 7 位设备地址
        ↓
发送 1 位读写位（0=写, 1=读）
        ↓
等待从机应答（ACK）
        ↓
    数据传输
        ↓
主机发送 STOP 条件
```

## 3. 硬件准备

### 3.1 所需材料

| 序号 | 名称 | 数量 | 说明 |
|------|------|------|------|
| 1 | ESP32-S3R8N8 开发板 | 1 | 嘉立创出品 |
| 2 | I2C 传感器模块 | 1+ | 如 OLED、温湿度传感器等 |
| 3 | 杜邦线 | 若干 | 公对母或母对母 |
| 4 | USB 数据线 | 1 | Type-C 接口 |

### 3.2 接线图

```
ESP32-S3 开发板          I2C 设备
┌─────────────┐         ┌─────────────┐
│             │         │             │
│         GPIO4 ├───────┤ SDA         │
│             │         │             │
│         GPIO5 ├───────┤ SCL         │
│             │         │             │
│          3.3V ├───────┤ VCC         │
│             │         │             │
│           GND ├───────┤ GND         │
│             │         │             │
└─────────────┘         └─────────────┘
```

**接线说明：**
- **SDA（数据线）**：ESP32 GPIO4 → 传感器 SDA
- **SCL（时钟线）**：ESP32 GPIO5 → 传感器 SCL
- **VCC（电源）**：3.3V（注意电压匹配！）
- **GND（地线）**：共地连接

### 3.3 注意事项

⚠️ **重要提醒：**

1. **电压匹配**：ESP32-S3 工作电压为 3.3V，确保传感器支持 3.3V
2. **上拉电阻**：大多数模块已内置上拉电阻，无需外接
3. **电源滤波**：建议在 VCC 和 GND 之间加 100nF 电容
4. **线材长度**：I2C 总线建议不超过 1 米

## 5. 代码详解

### 5.1 主程序入口 ([main.rs](src/bin/main.rs))

```rust
#![no_main]  // 告诉编译器：没有标准的 main 函数入口
#![no_std]   // 告诉编译器：不使用标准库（嵌入式开发必须）

use esp_hal::{delay::Delay, i2c::master::I2c, main};
use panic_rtt_target as _;
use rtt_target::{rprint, rprintln, rtt_init_print};
```

**关键点：**
- `#![no_std]`：嵌入式环境没有操作系统，不能使用标准库
- `#![no_main]`：ESP32 使用自己的启动流程
- `rtt_target`：通过 RTT（Real-Time Transfer）输出调试信息

### 5.2 常量定义

```rust
/// I2C 扫描的起始地址（0x08 是有效地址的起始）
const SCAN_START: u8 = 0x08;

/// I2C 扫描的结束地址（0x78 是保留地址的起始）
const SCAN_END: u8 = 0x78;
```

**为什么选择 0x08-0x77？**
- 0x00-0x07：保留用于特殊用途（广播、10 位地址等）
- 0x08-0x77：标准 7 位 I2C 地址范围
- 0x78-0x7F：保留用于 10 位地址扩展

### 5.3 设备识别函数

```rust
fn device_name(addr: u8) -> Option<&'static str> {
    match addr {
        0x3C => Some("OLED SSD1306 (128x32)"),
        0x76..=0x77 => Some("Env Sensor (BME280/BMP280)"),
        // ... 更多设备
        _ => None,
    }
}
```

**功能说明：**
- 输入：7 位 I2C 地址
- 输出：设备名称（如果识别成功）
- 用途：帮助开发者快速识别扫描到的设备

### 5.4 I2C 初始化

```rust
let mut i2c = I2c::new(peripherals.I2C0, esp_hal::i2c::master::Config::default())
    .unwrap()
    .with_sda(peripherals.GPIO4)  // 数据线连接到 GPIO4
    .with_scl(peripherals.GPIO5); // 时钟线连接到 GPIO5
```

**参数说明：**
- `peripherals.I2C0`：使用第一个 I2C 控制器
- `Config::default()`：默认配置（100kHz 标准模式）
- `GPIO4/5`：指定 SDA 和 SCL 引脚

### 5.5 扫描逻辑

```rust
// 尝试向设备发送数据来检测设备是否存在
let data: [u8; 1] = [0];
match i2c.write(addr, &data) {
    Ok(_) => {
        // 设备应答，打印地址
        rprint!("{:02X}", addr);
        found_count += 1;
    }
    Err(_) => {
        // 设备未应答，打印占位符
        rprint!("..");
    }
}
```

**工作原理：**
1. 向目标地址发送 1 字节数据（0x00）
2. 如果设备存在，会发送 ACK 应答信号
3. 如果设备不存在，会超时或返回 NAK

### 5.6 结果输出

```rust
// 打印设备详细信息
rprint!("  [{}] 0x{:02X}", i + 1, addr);
rprint!(" (W:0x{:02X} R:0x{:02X})", addr << 1, (addr << 1) | 1);
if let Some(name) = device_name(addr) {
    rprint!(" {}", name);
}
```

**输出格式：**
```
[序号] 7位地址 (写地址, 读地址) 设备名称
示例：[1] 0x3C (W:0x78 R:0x79) OLED SSD1306 (128x32)
```

## 9. 常见问题

### 9.1 编译错误

**问题：** `error[E0463]: can't find crate for 'std'`

**解决：**
```bash
# 确保已添加 ESP32-S3 目标
rustup target add xtensa-esp32s3-none-elf
```

### 9.2 烧录失败

**问题：** `Error: Failed to connect to the device`

**解决：**
1. 检查 USB 线是否支持数据传输（非充电线）
2. 按住 BOOT 按钮再插入 USB
3. 检查串口权限（Linux）：
   ```bash
   sudo usermod -a -G dialout $USER
   ```

### 9.3 扫描不到设备

**问题：** 所有地址都显示 `..`

**排查步骤：**
1. ✅ 检查接线是否正确（SDA、SCL 不要接反）
2. ✅ 检查设备是否上电（VCC、GND 是否连接）
3. ✅ 检查电压是否匹配（3.3V vs 5V）
4. ✅ 尝试外接上拉电阻（4.7kΩ 到 3.3V）
5. ✅ 降低 I2C 速度测试

### 9.4 RTT 输出乱码

**问题：** 串口输出乱码或无输出

**解决：**
```bash
# 安装正确的 RTT 监控工具
cargo install probe-rs --features cli

# 使用 probe-rs 监控
probe-rs attach --chip esp32s3
```
