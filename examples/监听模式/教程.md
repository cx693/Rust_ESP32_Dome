## 📖 前言

本篇学习 ESP32-S3 的 **WiFi 混杂模式（Promiscuous Mode）**，将网卡置于 Sniffer 状态，捕获空中的 802.11 Beacon 帧，提取并打印附近 AP 的 SSID。

- `no_std` 环境下的堆分配与集合操作

## 📡 原理说明

### WiFi 混杂模式

```
正常模式:                     混杂模式:
  网卡只接收发给自己的帧        网卡接收所有 WiFi 帧
  ┌─────┐                    ┌─────┐
  │ AP1 │──→ 我 ✓            │ AP1 │──→ 我 ✓
  │ AP2 │──→ 别人 ✗           │ AP2 │──→ 别人 ✓  ← 也能收到！
  │ AP3 │──→ 别人 ✗           │ AP3 │──→ 别人 ✓
  └─────┘                    └─────┘
```

### Beacon 帧

AP（路由器）每隔约 100ms 广播一次 Beacon 帧，包含：
- SSID（WiFi 名称）
- 支持的速率
- 安全类型等

## 📝 核心代码讲解

### 一、异步运行时初始化

WiFi 驱动需要 Embassy 异步运行时支持：

```rust
#[esp_rtos::main]
async fn main(_spawner: embassy_executor::Spawner) -> ! {
    // 1. 初始化硬件（CPU 最高频率 240MHz）
    let config = esp_hal::Config::default().with_cpu_clock(CpuClock::max());
    let peripherals = esp_hal::init(config);

    // 2. 分配堆内存（用于存储 SSID 字符串）
    esp_alloc::heap_allocator!(size: 72 * 1024);

    // 3. 启动 Embassy 运行时
    let timg0 = TimerGroup::new(peripherals.TIMG0);
    let sw_int = SoftwareInterruptControl::new(peripherals.SW_INTERRUPT);
    esp_rtos::start(timg0.timer0, sw_int.software_interrupt0);
}
```

### 二、WiFi Sniffer 配置

```rust
// 创建 WiFi 驱动
let (_controller, interfaces) =
    esp_radio::wifi::new(peripherals.WIFI, Default::default()).unwrap();

// 获取 sniffer 接口
let mut sniffer = interfaces.sniffer;

// 开启混杂模式（接收所有 WiFi 帧）
sniffer.set_promiscuous_mode(true).unwrap();
```

### 三、注册回调函数

```rust
// 全局状态：已发现的 SSID 集合（自动去重）
static KNOWN_SSIDS: Mutex<RefCell<BTreeSet<String>>> =
    Mutex::new(RefCell::new(BTreeSet::new()));

// 注册回调：每收到一帧 WiFi 数据就会被调用
sniffer.set_receive_cb(|packet| {
    // 尝试将原始字节解析为 BeaconFrame
    let _ = match_frames! {
        packet.data,
        beacon = BeaconFrame => {
            // 提取 SSID
            let Some(ssid) = beacon.ssid() else { return };

            // 临界区保护：安全访问全局数据
            if critical_section::with(|cs| {
                KNOWN_SSIDS.borrow_ref_mut(cs).insert(ssid.to_string())
            }) {
                // 只有新发现的 AP 才打印
                println!("Found new AP: {ssid}");
            }
        }
    };
});
```

### 四、临界区保护

```
中断回调 vs 主任务:
  ┌─────────────┐     ┌─────────────┐
  │  主任务      │     │  WiFi 中断   │
  │  读取 SSID   │ ←─→ │  写入 SSID   │
  │             │     │             │
  └─────────────┘     └─────────────┘
  
  问题: 同时读写会导致数据竞争
  
  解决: critical_section::with() 临时关中断
  ┌─────────────┐     ┌─────────────┐
  │  主任务      │     │  WiFi 中断   │
  │  ┌───────┐  │     │  (等待)      │
  │  │临界区  │  │     │             │
  │  │读写数据│  │     │             │
  │  └───────┘  │     │  ┌───────┐  │
  │             │     │  │临界区  │  │
  │             │     │  │读写数据│  │
  │             │     │  └───────┘  │
  └─────────────┘     └─────────────┘
```

## 🔍 知识点总结

| 知识点 | 说明 |
|--------|------|
| 混杂模式 | 接收所有 WiFi 帧，不仅是发给自己的 |
| Beacon 帧 | AP 广播帧，包含 SSID 等信息 |
| Embassy | Rust 异步运行时，适合嵌入式 |
| 临界区 | `critical_section::with()` 保护共享数据 |
| BTreeSet | 有序集合，自动去重 |
| match_frames! | 802.11 帧解析宏 |
